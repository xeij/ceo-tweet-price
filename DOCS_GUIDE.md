# Documentation Guide

This project includes several documentation files. Here's what each one contains:

## For Users

**README.md** - Start here! Brief overview in easy-to-read paragraphs covering what the tool does, how to use it, and basic troubleshooting.

**QUICKSTART.md** - Detailed step-by-step guide to get running in 5 minutes. Includes API key setup, example commands, and comprehensive troubleshooting.

## For Developers

**DEVELOPMENT.md** - Complete developer guide with architecture details, testing strategy, code style guidelines, and project roadmap.

**PROJECT_SUMMARY.md** - Comprehensive technical overview of all components, design decisions, and implementation status.

## Quick Reference

- **Just want to run it?** → Read README.md, then QUICKSTART.md
- **Want to contribute?** → Read DEVELOPMENT.md
- **Want to understand the architecture?** → Read PROJECT_SUMMARY.md
- **API keys needed?** → See QUICKSTART.md Step 2
- **Build failing?** → See QUICKSTART.md Troubleshooting section
- **Want to customize?** → See DEVELOPMENT.md "Development" section

## File Locations

- Source code: `src/`
- Lean proofs: `lean/`
- Tests: `src/*/tests` modules
- Examples: See QUICKSTART.md "More Examples" section
